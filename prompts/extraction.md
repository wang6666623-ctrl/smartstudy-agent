# 知识提取提示词

## 使用场景
用于 `speed-read` Skill 中，从教学文档提取结构化知识点。

## 提示词模板

```
你是一个知识提取专家。请从以下文档内容中提取结构化知识点。

## 提取要求

### 1. 结构分析
首先分析文档的层级结构，识别：
- 一级标题（章节）：用 # 标记
- 二级标题（小节）：用 ## 标记
- 知识点边界：每个独立概念的起止位置

### 2. 对每个知识点，提取以下信息
- **名称**：简洁准确的概念名称
- **定义**：一句话精确定义（保留原文表述）
- **要点**：3-5个核心要点
- **示例**：文档中的示例或自拟的典型示例
- **难度**：简单/中等/困难
- **类型**：概念类/算法类/应用类/推导类

### 3. 关系标注
- **前置知识**：理解本知识点需要先掌握什么
- **后续关联**：本知识点是哪些内容的基础
- **易混淆概念**：与本知识点容易混淆的相关概念

### 4. 摘要生成
- 每个小节：50-100字摘要
- 每个章节：150-200字摘要
- 全文：300-500字总摘要

## 输出格式
以 Markdown 格式输出，每个知识点一个独立卡片，包含上述所有字段。

## 质量检查
- 每个知识点的定义是否准确？
- 要点是否覆盖了核心内容？
- 难度标注是否合理？
- 知识点之间的关系是否标注完整？

## 文档内容
{文档内容}
```
