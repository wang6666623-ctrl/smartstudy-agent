# 意图路由提示词

## 功能说明

意图路由模块负责分析用户输入的意图，将其映射到对应的能力模块。

## 路由规则

根据以下规则识别用户意图，返回对应的 Skill 名称：

### 优先级1：文件处理意图
**触发条件**：
- 用户提供了文件路径（如 `C:\...\xxx.pdf`）
- 用户提到"这个文件"、"这个课件"、"这份资料"
- 用户拖拽了文件

**目标 Skill**：`speed-read`

### 优先级2：知识问答意图
**触发条件**：
- 以疑问词开头："什么是"、"为什么"、"如何"、"怎么"、"解释一下"
- 包含"是不是"、"对不对"、"有什么区别"
- 明确请求知识性回答

**目标 Skill**：`smart-qa`

### 优先级3：规划请求意图
**触发条件**：
- 包含"计划"、"规划"、"安排"、"备考"、"复习"
- 包含时间约束（日期、天数、周数）
- 明确提到考试/测验准备

**目标 Skill**：`review-planner`

### 优先级4：笔记处理意图
**触发条件**：
- 包含"整理"、"笔记"、"总结"、"归纳"
- 用户发来大量零散文本
- 提到"闪卡"、"卡片"、"记忆"

**目标 Skill**：`note-organizer`

### 优先级5：时间管理意图
**触发条件**：
- 包含"作业"、"截止"、"DDL"、"deadline"
- 包含"什么时候交"、"哪天考"、"还有多久"
- 用户列举课程/任务并询问时间

**目标 Skill**：`ddl-tracker`

## 多意图处理

如果用户输入同时匹配多个规则（如"帮我整理笔记，然后制定复习计划"），按以下策略处理：
1. 将请求拆分为独立子任务
2. 逐一调用对应的 Skill
3. 最终整合结果返回给用户

## 歧义处理

当无法唯一确定意图时（置信度 < 0.7），返回可能的意图列表，并询问用户选择：
```
我理解你可能想要：
1. [意图一] — 具体可以帮你xxx
2. [意图二] — 具体可以帮你xxx
你想要的是哪一个？
```

## 上下文增强

路由时考虑对话上下文：
- 如果上一轮用户在讨论某课程，本轮问"这个"，则关联到该课程
- 如果用户刚上传了文件，后续问题默认与该文件相关
- 追踪用户当前活跃的"会话主题"，辅助意图判断
