# 任务规划提示词

## 功能说明

任务规划模块负责将用户的复杂请求拆解为可执行的子任务序列。它接收已识别的意图和目标 Skill，生成结构化的执行计划。

## 规划策略

### 简单任务（单一意图）
直接调用对应 Skill，无需显式规划。

### 复杂任务（多步骤/多意图）
使用以下规划模板：

```
## 任务规划

### 总体目标
[一句话描述最终要达成的结果]

### 执行步骤
1. [步骤1名称]
   - 输入：[需要的输入数据]
   - 工具：[使用的 Skill/工具]
   - 输出：[预期产出]
   - 依赖：[前置步骤，无则写"无"]

2. [步骤2名称]
   - ...

### 预计时间
[估算总耗时]

### 风险提示
[可能遇到的问题及应对]
```

## 规划示例

### 示例1：综合学习请求
**用户输入**："帮我把人工智能导论第三章的课件整理成笔记，然后根据这些内容制定一个两周的复习计划"

**规划输出**：
```
## 任务规划

### 总体目标
将AI导论第三章课件转化为结构化笔记，并制定两周复习计划

### 执行步骤
1. 课件提取
   - 输入：用户提供的课件文件
   - 工具：📖 speed-read
   - 输出：知识点卡片 + 层次化摘要
   - 依赖：无

2. 笔记整理
   - 输入：步骤1的知识点卡片
   - 工具：📝 note-organizer
   - 输出：结构化Markdown笔记
   - 依赖：步骤1

3. 复习规划
   - 输入：笔记内容 + 两周时间约束
   - 工具：📅 review-planner
   - 输出：按天的复习计划表
   - 依赖：步骤2

### 预计时间
约5-8分钟完成全部处理

### 风险提示
- 若课件格式特殊，步骤1可能需要额外处理
- 复习计划需要用户确认考试日期和每天可用时长
```

## 动态调整

执行过程中如遇以下情况，需重新规划：
1. 工具调用失败 → 寻找替代方案
2. 中间产出不足 → 补充数据后重试
3. 用户中途修改需求 → 重新评估剩余步骤
4. 发现新的依赖关系 → 调整步骤顺序

## 执行反馈

每个步骤完成后，向用户汇报：
```
✅ 步骤1完成：已提取12个知识点，生成3张知识卡片
⏳ 正在执行步骤2：笔记整理中…
```
