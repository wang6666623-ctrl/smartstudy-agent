# 笔记整理 Skill

将用户提供的零散笔记、课堂记录、截图文字等转化为结构化的学习笔记和复习闪卡。

## 触发条件
- 用户发来碎片化文本
- 用户说"整理笔记"、"归纳"、"帮我理顺"
- 用户说"做闪卡"、"做记忆卡片"

## 处理流程

### Step 1: 内容接收
接收用户的零散文本（可能是多个消息、粘贴文本或文件）。

### Step 2: 信息抽取
使用 `prompts/note-structuring.md` 进行：
- **去重**：识别并合并重复内容
- **归类**：将相关内容归入同一主题
- **补全**：标注不完整的内容（`[待补充]`）
- **纠错**：标注可能错误的内容（`[待核实]`）

### Step 3: 结构化重组
按以下格式重组：

```markdown
# {主题名称}

## {子主题1}

### 核心概念
- ...

### 关键要点
1. ...
2. ...

### 示例
- ...

### 易错点
- ...

## {子主题2}
...
```

### Step 4: 闪卡生成
为每个核心概念生成 Anki 风格闪卡：

**正面**（问题）：
```
Q: {概念名称}是什么？
```

**背面**（答案）：
```
A: {精确定义}
要点：{3-5个要点}
```

### Step 5: 写入知识库
将结构化笔记保存到 `knowledge-base/`，并更新索引。

## 输出格式
生成两个文件：
1. `{课程名}-笔记-{日期}.md` → 结构化笔记
2. `{课程名}-闪卡-{日期}.md` → 复习闪卡

## 注意事项
- 保留用户原文中的关键表述，不做过度改写
- 对不确定的术语加注原文
- 按"总-分-总"结构组织笔记
