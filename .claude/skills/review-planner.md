# 复习规划 Skill

根据课程知识体系、考试时间和用户掌握程度，自动生成科学的阶段性复习计划。

## 触发条件
- 用户说"复习计划"、"备考"、"帮我安排复习"
- 用户提供考试日期和课程信息
- 用户说"还有X天考试，怎么复习"

## 处理流程

### Step 1: 信息收集
从用户获取：
- 课程名称（如已记录则直接使用）
- 考试日期 / 目标日期
- 每天可用学习时长（默认2-3小时）
- 已掌握和薄弱的知识点（查 `memory/user-profile.json`）

### Step 2: 知识体系分析
从知识库获取课程知识结构：
- 读取 `knowledge-base/index.json` 中该课程的知识点
- 按依赖关系排序（前置知识先复习）
- 按难度和重要性分配权重

### Step 3: 计划生成
使用 `prompts/review-generation.md` 生成计划：

**时间分配算法**：
```
单个知识点复习时长 = 基础时长 × 难度系数 × (1 + 薄弱系数)
总复习轮次 = ⌈总知识点 / 每天可学知识点⌉
每天分配 = 按优先级排序后平均分配，留最后2天总复习
```

**阶段划分**：
1. **基础巩固期**（前40%天数）：逐章复习，确保每个知识点都过一遍
2. **重点突破期**（中间30%天数）：集中攻克难点和薄弱环节
3. **综合演练期**（后20%天数）：做综合性题目，模拟考试
4. **最后冲刺期**（最后10%天数）：查漏补缺，回顾错题

### Step 4: 输出计划表
生成 Markdown 格式的计划表，写入文件并返回用户。

## 输出示例

```markdown
# 📅 《人工智能导论》两周复习计划

**考试日期**：2026-07-22
**总复习天数**：14天
**每日时长**：2.5小时

## 第一阶段：基础巩固（Day 1-6）

| 日期 | 章节 | 知识点 | 预计时长 | 优先级 |
|------|------|--------|---------|--------|
| 7/8 | 第1章 绪论 | 人工智能定义、发展史、应用领域 | 2h | ★★☆ |
| 7/9 | 第2章 搜索 | 无信息搜索(BFS/DFS) | 2.5h | ★★★ |
| ... | ... | ... | ... | ... |

## 第二阶段：重点突破（Day 7-10）
...
```

## 自适应调整
- 如果用户反馈某天计划未完成，自动重新分配剩余天数
- 如果用户某知识点掌握超预期，减少该知识点后续复习时间
