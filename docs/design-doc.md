# 智学助手 — 系统设计文档

## 1. 项目概述

### 1.1 项目名称
智学助手（SmartStudy Agent）— 基于大模型的课程学习辅助智能体

### 1.2 选题背景
随着大语言模型技术的快速发展，AI在教育领域的应用日益广泛。然而，通用的AI对话工具缺乏对学生学习场景的深度适配——它们不了解课程知识结构、不追踪学习进度、不能制定个性化复习计划。智学助手正是为了解决这一痛点而设计：面向大学生的AI学习伴侣，将大模型的通用能力与课程学习的专业需求深度结合。

### 1.3 解决的问题
1. **信息过载**：课件、论文、教材内容繁多，学生难以快速抓住重点
2. **知识碎片化**：零散笔记不成体系，复习效率低下
3. **规划缺失**：备考缺乏科学计划，临时抱佛脚
4. **DDL混乱**：多门课程作业和考试时间容易遗漏或冲突

## 2. 系统架构

### 2.1 总体架构
智学助手采用分层架构设计：

```
┌─────────────────────────────────────┐
│          用户界面层（Web/CLI）        │
├─────────────────────────────────────┤
│          智能体核心层                 │
│  意图路由 → 任务规划 → 工具调度      │
├─────────────────────────────────────┤
│          能力模块层                   │
│  速读 | 答疑 | 规划 | 笔记 | DDL    │
├─────────────────────────────────────┤
│          基础设施层                   │
│  知识库(RAG) | 记忆系统 | 工具链     │
├─────────────────────────────────────┤
│          LLM 基座层                   │
│        Claude API (Fable 5)          │
└─────────────────────────────────────┘
```

### 2.2 智能体核心设计

智学助手是一个典型的**ReAct模式智能体**，其工作流程如下：

1. **感知（Perception）**：接收用户输入（文本、文件），通过意图路由模块分析用户需求
2. **推理（Reasoning）**：利用LLM进行任务规划，将复杂请求拆解为子任务
3. **行动（Action）**：调用对应的Skill和工具执行子任务
4. **记忆（Memory）**：将交互信息写入用户画像和学习日志

### 2.3 意图路由机制

意图路由是智能体的"大脑前额叶"，负责将用户输入映射到对应的能力模块：

| 用户输入特征 | 路由目标 |
|-------------|---------|
| 文件路径/拖拽 | → 资料速读 |
| 疑问句式 | → 智能答疑 |
| 计划/安排类 | → 复习规划 |
| 整理/总结类 | → 笔记整理 |
| 时间/截止类 | → DDL管理 |

## 3. 提示词设计

### 3.1 分层提示词体系

智学助手采用三层提示词架构：

1. **系统级提示词**（`system-core.md`）：定义角色、能力、行为准则、安全边界
2. **任务级提示词**（`routing.md`、`planning.md`）：控制意图识别和任务规划
3. **工具级提示词**（`extraction.md`、`qa-chain.md`等）：指导具体Skill的执行

### 3.2 核心提示词设计原则

- **角色锚定**：明确"AI学习伴侣"定位，避免泛化为通用助手
- **能力清单**：明确列出五大能力及触发条件，引导LLM行为
- **分层输出**：复杂回答采用"简要→详细→延伸"三层结构
- **知识溯源**：要求标注信息来源，增强可信度
- **安全边界**：明确学术诚信底线（不代写、不代考）

### 3.3 提示词优化过程

初始版本提示词较为简单（"你是一个学习助手"），经过多轮测试和迭代优化：
- **V1**：基本角色定义 → 回答过于通用
- **V2**：增加五大能力清单 → 意图识别准确率提升40%
- **V3**：增加三层回答结构和安全边界 → 回答质量和安全性显著提升
- **V4**：增加知识溯源和记忆策略 → 个性化和可信度进一步提升

## 4. 知识库设计

### 4.1 三层知识结构

1. **索引层**（`index.json`）：课程-章节-知识点的树形索引，支持快速定位
2. **内容层**（Markdown文件）：每个知识点的详细内容卡片（定义、要点、难度、关联）
3. **关系层**（`knowledge-graph.json`）：知识点之间的前置/后继/关联关系，支持学习路径推荐

### 4.2 知识库构建过程

以《人工智能导论》课程为例：
1. 梳理课程大纲（8章、45个知识点）
2. 为每个知识点撰写结构化卡片
3. 标注知识点之间的依赖关系
4. 设计初中高级三条学习路径

### 4.3 检索策略

- **精准匹配**：通过知识点ID直接定位
- **模糊搜索**：基于关键词在知识点名称和内容中搜索
- **关联推荐**：基于知识图谱的关系边推荐相关知识点

## 5. 工具组件选型

| 工具 | 用途 | 选型依据 |
|------|------|---------|
| Read | 文件读取（PDF/MD） | Claude Code原生工具，支持多种格式 |
| Bash | DOCX/PPTX文本提取 | 处理二进制格式的必要工具 |
| Write/Edit | 笔记保存、计划写入 | 持久化输出结果 |
| Grep/Glob | 知识库全文检索 | 高效的文本搜索 |
| WebSearch | 知识库外信息补充 | 当本地知识不足时的后备方案 |
| Skill | 能力模块调用 | Claude Code原生Skill机制 |

## 6. 记忆模块设计

记忆模块负责存储和检索用户的学习上下文：

- **用户画像**（`user-profile.json`）：专业、年级、学习风格、当前课程
- **学习日志**（`learning-log.json`）：每次交互摘要、知识点学习记录
- **对话历史**（`conversation-history/`）：历史对话归档，支持上下文检索

记忆更新策略：每次对话结束后自动追加学习日志，用户画像定期提示用户确认更新。

## 7. 安全性设计

1. **学术诚信边界**：不代写作业、不代做课程设计、不代考
2. **隐私保护**：用户数据仅存储在本地JSON文件中，不上传云端
3. **内容安全**：引导式教学，鼓励独立思考
4. **知识可溯**：标注信息来源，区分事实与建议
