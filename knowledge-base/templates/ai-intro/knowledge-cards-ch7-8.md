## 第7章：计算机视觉

### KP-036: 计算机视觉概述
- **定义**：使计算机能从图像和视频中获取高层语义理解的技术领域
- **要点**：
  1. 核心任务：分类、检测、分割、跟踪、生成
  2. 传统方法：SIFT、HOG + SVM（手工特征时代）
  3. CNN革命（2012 AlexNet）开启了深度学习视觉时代
  4. 数据驱动：ImageNet等大规模数据集是关键推动力
- **难度**：中等
- **前置知识**：KP-027（CNN）
- **关联**：CNN(KP-027)、目标检测(KP-037)

### KP-037: 目标检测
- **定义**：同时定位和识别图像中多个目标的位置和类别
- **要点**：
  1. 两阶段方法：Faster R-CNN（RPN提出候选区域→分类+回归）
  2. 单阶段方法：YOLO（直接回归边界框和类别）
  3. YOLO速度快，适合实时应用；两阶段精度更高
  4. 核心指标：mAP（平均精度均值）、IoU（交并比）
- **难度**：困难
- **前置知识**：KP-027、KP-036
- **关联**：CNN(KP-027)

### KP-038: 图像分割
- **定义**：将图像中的每个像素分配到对应类别或实例
- **要点**：
  1. 语义分割：同一类别的所有物体归为一类（FCN、U-Net、DeepLab）
  2. 实例分割：区分同一类别中的不同实例（Mask R-CNN）
  3. U-Net结构：编码器-解码器 + 跳跃连接，适合医学图像
  4. 应用：自动驾驶（路面/行人分割）、医学影像
- **难度**：困难
- **前置知识**：KP-027、KP-036
- **关联**：CNN(KP-027)

### KP-039: 多模态模型
- **定义**：能够同时理解和处理多种模态信息（文本、图像、音频等）的模型
- **要点**：
  1. CLIP：通过对比学习对齐图像和文本的表示空间
  2. 零样本分类能力：不需要在特定类别上训练
  3. GPT-4V/Claude Vision：多模态大模型
  4. 图文生成：Stable Diffusion使用文本描述生成图像
  5. 应用：图文搜索、视觉问答、AI绘画
- **难度**：中等
- **前置知识**：KP-028、KP-030
- **关联**：Transformer(KP-028)、扩散模型(KP-030)

## 第8章：大模型与AIGC

### KP-040: GPT系列技术演进
- **定义**：OpenAI开发的生成式预训练Transformer系列模型的技术发展路径
- **要点**：
  1. GPT-1（2018）：证明了"预训练+微调"范式的有效性
  2. GPT-2（2019）：展示了零样本迁移能力
  3. GPT-3（2020）：175B参数，涌现出上下文学习能力
  4. GPT-4（2023）：多模态能力，更强的推理和安全性
  5. 技术趋势：更大的模型+更多的数据→更强的涌现能力
- **难度**：中等
- **前置知识**：KP-028、KP-031
- **关联**：LLM(KP-031)、Transformer(KP-028)

### KP-041: 提示工程 (Prompt Engineering)
- **定义**：设计和优化输入提示以引导大语言模型产生期望输出的技术和实践
- **要点**：
  1. 零样本提示(Zero-shot)：直接提问，不给示例
  2. 少样本提示(Few-shot)：提供2-5个示例，模型模仿回答模式
  3. 思维链(Chain-of-Thought)：提示中加入推理步骤"Let's think step by step"
  4. 角色扮演提示：设定AI角色（"你是一个专业的…"）
  5. 结构化输出：指定输出格式（JSON、Markdown表格等）
- **难度**：中等
- **前置知识**：KP-031
- **关联**：LLM(KP-031)、AI Agent(KP-042)

### KP-042: AI Agent（人工智能智能体）
- **定义**：能够自主感知环境、制定计划、调用工具并执行行动来完成目标的AI系统
- **要点**：
  1. 核心组成：LLM大脑 + 工具调用 + 记忆系统 + 规划模块
  2. ReAct模式：Reasoning（推理）+ Acting（行动）交替进行
  3. 工具使用：搜索、代码执行、API调用、数据库查询
  4. 记忆类型：短期记忆（对话上下文）和长期记忆（外部存储）
  5. 多Agent协作：多个Agent分工协作完成复杂任务
- **难度**：困难
- **前置知识**：KP-031、KP-041
- **关联**：LLM(KP-031)、提示工程(KP-041)、RAG(KP-035)

### KP-043: AIGC应用生态
- **定义**：利用AI生成文本、图像、音频、视频、代码等内容的工具和应用生态
- **要点**：
  1. 文本生成：ChatGPT、Claude、Kimi、文心一言
  2. 图像生成：Midjourney、DALL-E、Stable Diffusion
  3. 代码生成：GitHub Copilot、Cursor、Claude Code
  4. 视频生成：Sora、Runway、可灵
  5. 行业应用：教育、医疗、设计、娱乐、制造业
- **难度**：简单
- **前置知识**：KP-040
- **关联**：GPT(KP-040)、AI Agent(KP-042)

### KP-044: 大模型安全与伦理
- **定义**：在大模型开发和应用中需要关注的安全性、公平性、隐私和伦理问题
- **要点**：
  1. 幻觉问题：LLM可能生成看似合理但实际错误的内容
  2. 偏见与公平：训练数据中的社会偏见可能被模型放大
  3. 隐私泄露：训练数据可能包含个人隐私信息
  4. 对抗攻击：精心设计的提示可能绕过安全限制(Jailbreak)
  5. 对齐(Alignment)：RLHF等技术使模型行为与人类价值观一致
  6. AI监管：各国AI法规（欧盟AI Act、中国的生成式AI管理办法）
- **难度**：中等
- **前置知识**：KP-031
- **关联**：LLM(KP-031)、AI Agent(KP-042)

### KP-045: 知识图谱
- **定义**：以图结构组织知识的语义网络，节点代表实体/概念，边代表关系
- **要点**：
  1. 三元组：(头实体, 关系, 尾实体) 如 (SCUT, located_in, 广州)
  2. 通用知识图谱：Wikidata、DBpedia、CN-DBpedia
  3. 领域知识图谱：针对特定行业构建的专业图谱
  4. 与大模型的结合：KG增强的RAG、KG辅助推理
  5. 构建流程：实体识别→关系抽取→实体消歧→图谱补全
- **难度**：中等
- **前置知识**：KP-013
- **关联**：语义网络(KP-013)、RAG(KP-035)
