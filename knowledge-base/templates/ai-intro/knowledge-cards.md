## 第1章：绪论

### KP-001: 人工智能的定义
- **定义**：人工智能（Artificial Intelligence, AI）是研究、开发用于模拟、延伸和扩展人的智能的理论、方法、技术及应用系统的一门新的技术科学
- **要点**：
  1. AI的目标是让机器能像人一样思考、学习、决策
  2. 强AI vs 弱AI：强AI具备通用智能，弱AI专注于特定任务
  3. 图灵测试是判断机器是否具有智能的经典方法
  4. 当前AI主要是弱AI（狭义AI）
- **难度**：简单
- **前置知识**：无
- **关联**：图灵测试(KP-002)、AI发展史(KP-003)

### KP-002: 图灵测试
- **定义**：由艾伦·图灵于1950年提出的判断机器智能的方法——如果一台机器能够与人类对话而不被辨别出机器身份，则被认为具有智能
- **要点**：
  1. 测试设置：人类评判员通过文本与机器和人类交流
  2. 评判员需要判断哪个是机器
  3. 机器若能欺骗30%以上的评判员，即通过测试
  4. 局限性：只测试对话能力，不测试其他形式的智能
- **难度**：简单
- **前置知识**：KP-001
- **关联**：中文房间实验(KP-004)

### KP-003: 人工智能发展简史
- **定义**：AI从诞生至今经历了多次高潮与低谷的发展历程
- **要点**：
  1. 1956年达特茅斯会议：AI正式诞生
  2. 第一次AI浪潮（1956-1974）：符号主义主导，逻辑推理
  3. 第一次AI寒冬（1974-1980）：算力不足，预期过高
  4. 专家系统热潮（1980-1987）：知识工程的兴起
  5. 第二次AI寒冬（1987-1993）：专家系统局限暴露
  6. 深度学习革命（2012至今）：大数据+GPU推动AI爆发
- **难度**：简单
- **前置知识**：KP-001
- **关联**：深度学习(KP-024)

### KP-004: 中文房间实验
- **定义**：约翰·塞尔提出的思想实验，质疑"通过图灵测试就是有智能"的观点——一个不懂中文的人按照规则书处理中文符号，外面的人误以为他懂中文
- **要点**：
  1. 核心论点：语法不等于语义
  2. 对强AI的质疑：符号操作不产生真正的理解
  3. 反驳观点：系统整体可能具有理解(系统回应)
  4. 影响：引发关于AI是否能有"意识"的长期哲学讨论
- **难度**：中等
- **前置知识**：KP-002
- **关联**：图灵测试(KP-002)、知识表示(KP-011)

## 第2章：搜索技术

### KP-005: 无信息搜索（盲目搜索）
- **定义**：不利用问题域特定知识的搜索策略，仅根据图结构进行遍历
- **要点**：
  1. BFS（广度优先搜索）：逐层扩展，保证找到最浅解
  2. DFS（深度优先搜索）：沿一条路径深入，节省内存但可能陷入深分支
  3. 一致代价搜索（UCS）：扩展代价最小的节点，寻最优解
  4. 完备性 vs 最优性：BFS完备且最优（等代价），DFS不完备
- **难度**：中等
- **前置知识**：基础数据结构（图、队列、栈）
- **关联**：启发式搜索(KP-006)、A*算法(KP-007)

### KP-006: 启发式搜索
- **定义**：利用启发式信息（问题域知识）引导搜索方向，提高搜索效率
- **要点**：
  1. 启发式函数 h(n)：估算节点n到目标的最小代价
  2. 贪心搜索：每次扩展 h(n) 最小的节点，不一定最优
  3. 启发式设计需领域知识
  4. 好的启发式能指数级缩小搜索空间
- **难度**：中等
- **前置知识**：KP-005
- **关联**：A*算法(KP-007)、启发式函数设计(KP-008)

### KP-007: A* 搜索算法
- **定义**：结合了实际代价 g(n) 和启发式估计 h(n) 的最优优先搜索算法，f(n) = g(n) + h(n)
- **要点**：
  1. f(n) 是经过节点n的最优路径的估计总代价
  2. 当 h(n) ≤ h*(n)（可纳），A*保证找到最优解
  3. 当 h(n) 满足一致性（单调性），A*不会重复扩展节点
  4. A*是最优的：在可纳启发式下，不会扩展比必要更多的节点
- **难度**：困难
- **前置知识**：KP-005, KP-006
- **关联**：IDA* (KP-009)、启发式函数设计(KP-008)

### KP-008: 启发式函数设计
- **定义**：为具体问题设计有效的启发式估计函数的方法
- **要点**：
  1. 松弛法：忽略部分约束后的问题最优解作为启发式
  2. 模式数据库：预计算子问题的精确解
  3. 多个启发式的组合：h(n) = max(h1(n), h2(n), ...)
  4. 曼哈顿距离（八数码）、直线距离（路径规划）
- **难度**：中等
- **前置知识**：KP-006
- **关联**：A*算法(KP-007)

### KP-009: IDA* 与迭代加深搜索
- **定义**：结合深度优先搜索的内存优势和A*的最优性的算法，通过不断增大的f值阈值进行深度优先搜索
- **要点**：
  1. 每次DFS限制f(n)不超过当前阈值
  2. 如果找不到解，阈值设为超过当前阈值的最小f值
  3. 内存使用与DFS相当（线性）
  4. 适用于状态空间大、内存受限的问题
- **难度**：困难
- **前置知识**：KP-007
- **关联**：A*算法(KP-007)

### KP-010: 对抗搜索（博弈树搜索）
- **定义**：在有对手的环境中进行搜索决策，典型应用于双人零和博弈
- **要点**：
  1. Minimax算法：MAX方最大化收益，MIN方最小化收益
  2. Alpha-Beta剪枝：剪除不影响最终决策的分支
  3. α-β剪枝可将时间复杂度从O(b^d)降至O(b^(d/2))（最优情况）
  4. 评估函数：在非终局状态估计局面优劣
- **难度**：困难
- **前置知识**：KP-005
- **关联**：蒙特卡洛树搜索

## 第3章：知识表示与推理

### KP-011: 知识表示概述
- **定义**：研究如何将人类知识形式化表示为计算机可以存储和处理的结构
- **要点**：
  1. 知识表示是AI系统的核心组件
  2. 好的表示应具备：表达充分性、推理有效性、操作高效性、获取容易性
  3. 声明式 vs 过程式知识
  4. 主要方法：逻辑、产生式、语义网络、框架、本体
- **难度**：中等
- **前置知识**：KP-001
- **关联**：一阶逻辑(KP-012)、语义网络(KP-013)

### KP-012: 一阶谓词逻辑
- **定义**：使用谓词、量词、变量和连接词来表达知识和推理的形式化语言
- **要点**：
  1. 基本元素：常量、变量、谓词、函数、量词(∀, ∃)、连接词(∧, ∨, ¬, →)
  2. 表达能力远超命题逻辑
  3. 消解原理：机械化推理方法
  4. Prolog语言基于一阶逻辑的子集(Horn子句)
- **难度**：困难
- **前置知识**：KP-011、离散数学
- **关联**：知识图谱(KP-045)

### KP-013: 语义网络
- **定义**：用图结构表示知识，节点表示概念，边表示概念间的关系
- **要点**：
  1. 基本关系：IS-A（继承）、PART-OF（组成）、INSTANCE（实例）
  2. 继承推理：子节点自动继承父节点的属性
  3. 直观易理解，但缺乏形式化语义
  4. 知识图谱可视为语义网络的现代扩展
- **难度**：中等
- **前置知识**：KP-011
- **关联**：知识图谱(KP-045)

### KP-014: 产生式系统
- **定义**：基于IF-THEN规则的知识表示和推理系统
- **要点**：
  1. 组成：规则库、工作内存、推理引擎
  2. 推理引擎循环：匹配→选择→执行
  3. 前向链接（数据驱动）vs 后向链接（目标驱动）
  4. 专家系统是产生式系统的典型应用
- **难度**：中等
- **前置知识**：KP-011
- **关联**：专家系统(KP-015)

### KP-015: 专家系统
- **定义**：利用领域专家的知识来解决特定领域复杂问题的AI系统
- **要点**：
  1. 核心组成：知识库 + 推理引擎 + 解释模块
  2. 知识获取是瓶颈（知识工程）
  3. 最早成功商用的AI技术之一（如MYCIN医疗诊断系统）
  4. 局限：知识脆弱、维护困难、缺乏学习能力
- **难度**：中等
- **前置知识**：KP-014
- **关联**：产生式系统(KP-014)

## 第4章：机器学习基础

### KP-016: 机器学习概述
- **定义**：机器学习是通过数据经验自动改进计算机算法性能的研究领域
- **要点**：
  1. 三大范式：监督学习、无监督学习、强化学习
  2. 核心三要素：数据、模型、优化算法
  3. 汤姆·米切尔的经典定义：从经验E中学习，在任务T上以性能度量P提升
  4. 与传统编程的区别：传统编程是"规则+数据→输出"，ML是"数据+答案→规则"
- **难度**：中等
- **前置知识**：KP-001
- **关联**：监督学习(KP-017)、无监督学习(KP-018)、强化学习(KP-019)

### KP-017: 监督学习
- **定义**：从带标签的训练数据中学习输入到输出的映射函数
- **要点**：
  1. 分类问题：输出是离散类别（垃圾邮件检测、图像识别）
  2. 回归问题：输出是连续值（房价预测、温度预测）
  3. 常见算法：线性回归、逻辑回归、SVM、决策树、随机森林
  4. 核心挑战：过拟合 vs 欠拟合
- **难度**：中等
- **前置知识**：KP-016
- **关联**：过拟合(KP-020)、模型评估(KP-021)

### KP-018: 无监督学习
- **定义**：从无标签数据中发现隐藏的结构和模式
- **要点**：
  1. 聚类：K-Means、DBSCAN、层次聚类
  2. 降维：PCA、t-SNE、自编码器
  3. 关联规则：Apriori算法
  4. 应用：客户分群、异常检测、数据可视化
- **难度**：中等
- **前置知识**：KP-016
- **关联**：监督学习(KP-017)

### KP-019: 强化学习
- **定义**：智能体通过与环境交互，试错学习最优行为策略
- **要点**：
  1. 核心要素：状态(State)、动作(Action)、奖励(Reward)、策略(Policy)
  2. 马尔可夫决策过程(MDP)是理论基础
  3. Q-Learning：学习状态-动作价值函数
  4. 深度强化学习：DQN、Policy Gradient、PPO
  5. 里程碑应用：AlphaGo、OpenAI Five
- **难度**：困难
- **前置知识**：KP-016
- **关联**：深度学习(KP-024)

### KP-020: 过拟合与正则化
- **定义**：过拟合是模型在训练数据上表现好但在新数据上泛化差的现象；正则化是防止过拟合的技术
- **要点**：
  1. 过拟合原因：模型过于复杂、训练数据不足或噪声大
  2. 检测方法：训练集和验证集性能差距大
  3. L1正则化(Lasso)：产生稀疏解
  4. L2正则化(Ridge)：限制权重大小
  5. Dropout：随机丢弃神经元，适用于神经网络
- **难度**：中等
- **前置知识**：KP-017
- **关联**：模型评估(KP-021)

### KP-021: 模型评估方法
- **定义**：评估机器学习模型性能的系统化方法
- **要点**：
  1. 训练集/验证集/测试集划分（通常70/15/15或80/10/10）
  2. K折交叉验证：小数据集的黄金标准
  3. 分类指标：准确率、精确率、召回率、F1、AUC-ROC
  4. 回归指标：MSE、RMSE、MAE、R²
  5. 混淆矩阵：可视化分类性能
- **难度**：中等
- **前置知识**：KP-017
- **关联**：过拟合(KP-020)

### KP-022: 决策树与随机森林
- **定义**：决策树是通过树形结构做决策的模型；随机森林是多棵决策树的集成
- **要点**：
  1. 决策树分裂准则：信息增益(ID3)、增益率(C4.5)、基尼系数(CART)
  2. 决策树易过拟合，需要剪枝
  3. 随机森林 = Bagging + 随机特征选择
  4. 随机森林可输出特征重要性
  5. 优点：可解释性强、无需特征缩放
- **难度**：中等
- **前置知识**：KP-017
- **关联**：监督学习(KP-017)、集成学习

### KP-023: 支持向量机(SVM)
- **定义**：通过寻找最大间隔超平面来进行分类的算法
- **要点**：
  1. 核心思想：寻找使两类样本间隔最大化的分隔超平面
  2. 核技巧：通过核函数隐式映射到高维空间（RBF核、多项式核）
  3. 软间隔：允许少量误分类，用参数C控制
  4. SVM可处理线性和非线性分类
  5. 适用于中小规模数据、高维特征
- **难度**：困难
- **前置知识**：KP-017、线性代数
- **关联**：监督学习(KP-017)

## 第5章：深度学习

### KP-024: 深度学习概述
- **定义**：使用多层神经网络从数据中进行层次化特征学习的机器学习子领域
- **要点**：
  1. 与传统ML区别：自动学习层次化特征表示，无需手工设计特征
  2. 三大驱动力：大数据、GPU算力、算法创新(ReLU/BatchNorm/ResNet等)
  3. 端到端学习：从原始输入到最终输出
  4. 表示学习：每一层学习越来越抽象的特征
- **难度**：中等
- **前置知识**：KP-016
- **关联**：神经网络基础(KP-025)、CNN(KP-027)、Transformer(KP-028)

### KP-025: 神经网络基础
- **定义**：受人脑神经元启发构建的计算模型，由神经元层和连接权重组成
- **要点**：
  1. 神经元 = 加权求和 + 激活函数
  2. 前向传播：输入→隐藏层→输出
  3. 反向传播：链式法则计算梯度
  4. 常用激活函数：ReLU、Sigmoid、Tanh、Softmax
  5. 损失函数：MSE(回归)、交叉熵(分类)
- **难度**：困难
- **前置知识**：KP-016、微积分
- **关联**：深度学习(KP-024)、优化算法(KP-026)

### KP-026: 优化算法
- **定义**：通过最小化损失函数来更新神经网络参数的算法
- **要点**：
  1. SGD：随机梯度下降，每次用一个小批量
  2. Momentum：积累历史梯度方向，加速收敛
  3. Adam：自适应学习率 + 动量，最常用的优化器
  4. 学习率调度：余弦退火、阶梯衰减、warmup
  5. Batch Normalization：加速训练，稳定梯度
- **难度**：困难
- **前置知识**：KP-025
- **关联**：神经网络(KP-025)

### KP-027: 卷积神经网络(CNN)
- **定义**：专门处理网格结构数据（如图像）的神经网络，使用卷积操作提取空间特征
- **要点**：
  1. 卷积层：局部感受野 + 权值共享
  2. 池化层：下采样，降低维度（MaxPooling、AvgPooling）
  3. 经典架构：LeNet → AlexNet → VGG → GoogLeNet → ResNet
  4. ResNet的残差连接解决了深层网络退化问题
  5. 应用：图像分类、目标检测、语义分割
- **难度**：困难
- **前置知识**：KP-025
- **关联**：深度学习(KP-024)、Transformer(KP-028)

### KP-028: Transformer架构
- **定义**：基于自注意力机制的序列建模架构，是现代大模型的基石
- **要点**：
  1. 自注意力(Self-Attention)：Q·K^T 计算序列内各位置的关联权重
  2. 多头注意力：多个注意力头并行，关注不同子空间
  3. 位置编码：弥补注意力机制对位置不敏感的缺陷
  4. 相比RNN的优势：并行计算、长程依赖建模
  5. "Attention Is All You Need" (2017) 开启了Transformer时代
- **难度**：困难
- **前置知识**：KP-025
- **关联**：大语言模型(KP-031)、BERT(KP-032)、GPT(KP-040)

### KP-029: 生成对抗网络(GAN)
- **定义**：由生成器和判别器组成，通过对抗训练来生成逼真数据的框架
- **要点**：
  1. 生成器G：从噪声生成假数据
  2. 判别器D：区分真假数据
  3. MiniMax博弈：min_G max_D V(D, G)
  4. 训练困难：模式崩塌、不收敛
  5. 应用：图像生成、风格迁移、数据增强
- **难度**：困难
- **前置知识**：KP-025
- **关联**：深度学习(KP-024)

### KP-030: 扩散模型
- **定义**：通过逐步添加和去除噪声来生成数据的生成模型
- **要点**：
  1. 前向过程：逐步向数据添加高斯噪声
  2. 反向过程：学习去噪，从纯噪声恢复数据
  3. DDPM、Stable Diffusion、DALL-E均基于此
  4. 相比GAN：训练更稳定，生成多样性更好
  5. 缺点：生成速度较慢（需多步采样）
- **难度**：困难
- **前置知识**：KP-025
- **关联**：GAN(KP-029)、AIGC(KP-043)

## 第6章：自然语言处理

### KP-031: 大语言模型(LLM)
- **定义**：基于大规模文本语料训练的巨型语言模型，展现出涌现能力
- **要点**：
  1. 扩展定律(Scaling Law)：模型性能随参数量、数据量、计算量幂律增长
  2. 涌现能力：上下文学习、思维链推理、代码生成
  3. 训练流程：预训练 → 指令微调 → RLHF/DPO
  4. 代表模型：GPT系列、Claude系列、LLaMA、Gemini
- **难度**：中等
- **前置知识**：KP-028
- **关联**：GPT(KP-040)、Transformer(KP-028)、提示工程(KP-041)

### KP-032: BERT与预训练
- **定义**：基于Transformer编码器的双向预训练语言模型
- **要点**：
  1. MLM（掩码语言模型）：随机mask部分token，预测被mask的词
  2. NSP（下一句预测）：判断两句话是否连续
  3. 双向上下文理解（不同于GPT的单向）
  4. 微调范式：预训练 + 下游任务微调
  5. 适用任务：文本分类、命名实体识别、问答
- **难度**：困难
- **前置知识**：KP-028
- **关联**：Transformer(KP-028)、GPT(KP-040)

### KP-033: 词向量与文本表示
- **定义**：将词语映射到低维稠密向量空间的技术
- **要点**：
  1. Word2Vec：CBOW和Skip-gram，通过预测任务学习词向量
  2. GloVe：基于全局词共现矩阵
  3. 语义关系：king - man + woman ≈ queen
  4. 局限：静态词向量无法处理多义词
  5. 上下文词向量（ELMo、BERT）动态解决多义问题
- **难度**：中等
- **前置知识**：KP-025
- **关联**：神经网络(KP-025)

### KP-034: 序列到序列模型(Seq2Seq)
- **定义**：将输入序列映射到输出序列的神经网络架构
- **要点**：
  1. 编码器：将输入序列压缩为上下文向量
  2. 解码器：从上下文向量生成输出序列
  3. 注意力机制：解码时动态关注编码器的不同部分
  4. 应用：机器翻译、文本摘要、对话生成
  5. Transformer完全取代了RNN-based Seq2Seq
- **难度**：中等
- **前置知识**：KP-025
- **关联**：Transformer(KP-028)

### KP-035: RAG与检索增强生成
- **定义**：结合信息检索和文本生成的混合架构，先从知识库检索相关文档再生成回答
- **要点**：
  1. 解决LLM的幻觉问题和知识截止日期限制
  2. 架构：Query → Retriever → Relevant Docs → Generator → Answer
  3. 检索技术：稠密检索(向量相似度) + 稀疏检索(BM25)
  4. 知识库可动态更新，无需重新训练模型
  5. 应用：企业知识问答、客服系统、研究助手
- **难度**：中等
- **前置知识**：KP-031、KP-033
- **关联**：LLM(KP-031)、知识图谱(KP-045)
